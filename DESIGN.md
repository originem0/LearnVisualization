# DESIGN.md — Learning Site Engine 设计规范 v3

> 这不是一份文件能装下的设计规范。这是入口。

## 这个系统在做什么

把复杂知识转化成可被理解、可被穿透、可被迁移的学习体验。不是内容工厂，不是好看的文章站，不是交互游乐场。

核心约束：**AI 辅助生成，人类把关质量，学习科学定规矩。**

## 核心设计张力

这个系统必须同时处理 5 对矛盾，每份设计文件都要回应它们：

1. **脚手架 vs 必要难度** — 新手需要支撑，但难度促进记忆。解法：渐退（fading）。
2. **静态导出 vs 自适应** — 没有服务器，但学习需要个性化。解法：客户端状态 + 条件渲染。
3. **通用引擎 vs 领域特化** — 系统 internals、代码、数学、设计各有教学传统。解法：领域教学法配置。
4. **结构约束 vs 创作自由** — Schema 太严会让 Agent 和作者写不出好内容。解法：必选字段最小化，扩展字段开放。
5. **质量自动检测 vs 教学判断** — 格式检查能自动化，但教学深度只能人判。解法：自动检查负责底线，人工审核负责上限。

## 非目标

1. 不做知识大超市 — 不是什么都能塞的集合站
2. 不做内容工厂 — 不追求大规模低质量自动生成
3. 不做纯文章站 — 不把"写得不错的长文"误当学习产品
4. 不做交互游乐场 — 交互必须服务认知目标
5. 不做全自动教学系统 — Agent 辅助生成，人类判断不可取消
6. 不做效率崇拜 — AI 放大判断力和创造力，不是加速产出垃圾

## 学习科学基础

详见 `design/01-learning-principles.md`。这里只列核心依据：

- **Bloom 修订分类学** (Anderson & Krathwohl 2001) — 知识维度 × 认知过程维度，决定教学策略选择
- **Merrill 教学首要原则** (2002) — 问题中心、激活、示范、应用、整合
- **认知负荷理论** (Sweller) — 内在/外在/相关负荷，元素交互性，worked example 效应
- **必要难度** (Bjork & Bjork) — 间隔、交错、检索练习、生成效应
- **多媒体学习** (Mayer) — 双通道、有限容量、一致性/信号/分段/空间邻近原则
- **认知学徒制** (Collins, Brown, Newman) — 建模/教练/脚手架/表达/反思/探索
- **4C/ID 模型** (van Merriënboer) — 整体任务/支持信息/即时信息/部分练习
