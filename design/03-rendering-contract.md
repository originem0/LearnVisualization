# 03. 渲染契约 — 不同知识怎么呈现

> 这份文件定义前端渲染引擎如何根据知识类型、模块类型、教学法做差异化呈现。它是 UI 工程师的约束文档。

---

## 一、核心原则：渲染跟着教学走

渲染引擎不是"把 JSON 块按顺序画出来"。它是教学策略的视觉实现。

**渲染决策树**：
```
moduleKind → 决定整体布局骨架
  ├── domainPedagogy → 决定内容阶段序列
  │     ├── PRIMM → predict / run / investigate / modify / make 五阶段
  │     ├── CRA → concrete / representational / abstract 三层
  │     └── cognitive-apprenticeship → model / coach / scaffold / articulate / reflect
  ├── knowledgeTypes → 决定侧边栏和练习类型
  └── elementInteractivity → 决定脚手架强度和分段粒度
```

---

## 二、按 ModuleKind 差异化布局

### concept-clarification（概念澄清）
- 误解开场强调（红色边框、全宽卡片）
- 对比块突出显示（并排双栏，不是上下堆叠）
- 概念侧边栏常驻，每个概念带"说出区别"交互
- 检索练习偏重分类和边界判断

### mechanism-walkthrough（机制追踪）
- steps 块是视觉主轴（居中，大号，可交互推进）
- 每步之间有状态快照显示
- 概念侧边栏只在首次出现时展开，后续折叠
- trace 块带执行状态表

### system-overview（系统全景）
- 概念图先于正文（全宽渲染）
- 叙事按层级展开（先全景 → 再逐层）
- 支持"缩放"交互：点击概念图节点跳到对应段落

### code-lab（代码实验）
- 左右分栏：左代码 / 右解释
- PRIMM 阶段标签显示当前处于哪个阶段
- trace 块自动高亮当前执行行
- 输出预测交互在代码块上方

### derivation（推导证明）
- 逐行推导，每行可展开理由
- CRA 三层表示可切换
- 符号定义悬浮提示
- 推导完成后回溯总结

### case-study（案例分析）
- 渐进揭示：场景 → 决策 → 后果，用户控制节奏
- 专家标注在侧边或内联批注
- 多视角对比（如果有多个案例）

### design-critique（设计批评）
- 双栏：设计作品 / 批评框架
- 标注层叠加在设计作品上
- 迭代时间线显示（版本 1 → 反馈 → 版本 2）

### meta-reflection（元认知反思）
- 不追加新知识，全部是回溯和重建
- 概念图显示为"空白版"，学习者自己填充
- 反思提示是主轴，不是叙事

### integration-review（系统整合）
- 多模块概念图合并显示
- 跨模块关系高亮
- 练习以"跨概念推理"为主

## 三、叙事块渲染规则

### 基础块（v2 保留）

| 类型 | 渲染方式 |
|------|---------|
| heading | `<h2>` / `<h3>`，生成 TOC 锚点 |
| text | 正文段落，支持多段 |
| code | 语法高亮，行号可选 |
| diagram | 居中块，带可选 label |
| comparison | 双栏并排（不是上下） |
| callout | 强调色背景卡片 |
| steps | 用户控制推进，每步带 visual + highlight |

### 语义块（v3 新增）

| 类型 | 渲染方式 |
|------|---------|
| annotated-example | 代码/文本 + 侧边专家批注 |
| trace | 状态表 + 当前步高亮 + 上下步导航 |
| reflection | 黄色背景卡片，带"我卡住了"/"我准备好了"两个按钮 |
| expert-thought | 灰色气泡样式（像思维过程），斜体 |
| generation | 输入框 + 提交 + 参考答案折叠 |
| analogy | 双栏：来源域 / 目标域，连线显示映射 |

### 渲染约束（来自 Mayer 原则）

1. **diagram 后必须有 text** — 不允许孤立图表
2. **code 和 text 必须空间邻近** — 不允许隔 3 个块以上
3. **非必要块 (isEssential=false) 默认折叠** — 点击展开
4. **spatiallyLinkedTo 的两个块渲染为并排** — 不是上下堆叠
5. **steps 块默认用户控制推进** — 不自动播放

## 四、概念侧边栏重设计

当前：惰性词条列表。v3 要求：

1. **主动回忆模式**：点击概念名 → 隐藏定义 → 要求学习者先回忆 → 再揭示
2. **前置知识链接**：如果概念属于前序模块，显示"首次出现于 s02"链接
3. **掌握度指示**：根据客户端状态显示掌握度色条（绿/黄/红）
4. **随叙事滚动**：当前段落相关的概念自动高亮

## 五、练习渲染

### 脚手架级别

| 级别 | 显示 | 用户操作 |
|------|------|---------|
| full | 完整 worked example，全部步骤 | 阅读 + 自我解释 |
| faded-1 | 大部分步骤，最后 1-2 步空白 | 填写最后步骤 |
| faded-2 | 只给起始条件和目标 | 填写大部分步骤 |
| free | 只给问题 | 独立解决 |

### 反馈渲染

- 即时反馈：提交后立即显示（不等下一页）
- 错误反馈：不只说"错了"——展示具体哪里错，指向相关概念
- 提示递进：第一次不给提示，错了给 hint[0]，再错给 hint[1]

## 六、响应式策略

| 断点 | 布局变化 |
|------|---------|
| < 640px (mobile) | 单栏，概念侧边栏移到底部，comparison 变上下 |
| 640-1024px (tablet) | 叙事居中，侧边栏折叠为浮动按钮 |
| 1024-1280px (laptop) | 叙事 + 侧边栏双栏 |
| > 1280px (desktop) | 叙事 + 侧边栏 + 浮动 TOC 三栏 |

## 七、设计语言

### 色彩语义

| 用途 | 色彩 | 说明 |
|------|------|------|
| 误解/危险 | red-50 / red-200 | 误解开场、陷阱标记 |
| 洞察/成功 | emerald-50 / emerald-200 | keyInsight、正确反馈 |
| 反思/元认知 | amber-50 / amber-200 | reflection 块、元认知提示 |
| 代码/技术 | slate-50 / slate-200 | code 块、trace 块 |
| 交互/行动 | blue-500 / blue-600 | 按钮、链接、进度 |
| 专家思维 | gray-100 | expert-thought 块 |

### 排版

- 正文：16px / 1.75 行高 / 最大宽度 65ch
- 标题：不超过 3 级（h1 页标题、h2 小节、h3 子节）
- 代码：14px / monospace / 行号

### 动效

- 练习提交：正确 → 向上滑出绿色确认；错误 → 左右抖动
- steps 推进：下一步从右侧滑入
- 概念揭示：从模糊到清晰（0.3s ease）
- 不用装饰性动画
