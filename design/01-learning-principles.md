# 01. 学习原则 — 系统设计的硬约束

> 这份文件不是文献综述。它从学习科学中提取对系统设计有直接约束力的原则，每条原则都指向一个必须在 schema、渲染或 Agent prompt 中落地的设计决策。

---

## 一、知识不止一种，教法必须跟着变

### 1.1 知识类型分类（Bloom 修订版 + De Jong）

| 类型 | 定义 | 教学需要 | 系统要求 |
|------|------|---------|---------|
| 事实性 (Factual) | 术语、细节、约定 | 间隔重复、标签练习 | 闪卡交互、fill-blank |
| 概念性 (Conceptual) | 分类、原则、关系 | 对比、分类、概念图 | comparison 块、concept map、classify 练习 |
| 程序性 (Procedural) | 算法、技巧、操作序列 | Worked example → 渐退 → 自主 | steps 块 + fading 变体 |
| 策略性 (Strategic) | 问题识别、方案选择 | 案例推理、专家思维外显 | expert-thought 块、case-study 模块 |
| 元认知 (Metacognitive) | 自我监控、策略评估 | 反思提示、自我解释 | reflection 块、metacognitive-checkpoint |
| 情境性 (Situational) | 问题类型识别 | 多案例对比、模式匹配 | 交错练习、多场景对比 |

**系统约束**：每个模块必须声明它教的是哪种知识类型。渲染引擎和 Agent 根据知识类型选择教学策略。不声明知识类型的模块不允许发布。

### 1.2 认知层级（Bloom 六级）

记忆 → 理解 → 应用 → 分析 → 评价 → 创造

**系统约束**：
- 课程内的模块序列应体现认知层级推进（不要求线性，但不能全停留在"记忆"层）
- 检索练习必须标注 Bloom 层级，质量检查拒绝全课程只有"记忆"层的检索
- Agent 生成的检索练习必须至少覆盖"分析"层

## 二、教学不止一种模式

### 2.1 Merrill 教学首要原则（通用基线）

所有模块必须满足的最低要求：

1. **问题中心** — 模块围绕一个真实问题展开，不是围绕术语定义
2. **激活** — 开场必须连接学习者已有知识（focusQuestion + misconception）
3. **示范** — 必须展示新知识如何运作（narrative + worked example）
4. **应用** — 必须让学习者应用新知识，且脚手架逐渐撤除
5. **整合** — 必须帮学习者把新知识接入已有体系（bridge + retrieval）

### 2.2 领域特化教学法

在 Merrill 基线之上，不同领域叠加专用教学法：

**代码领域 — PRIMM**
- 预测 (Predict)：给代码，猜输出
- 运行 (Run)：执行验证
- 探究 (Investigate)：修改观察变化
- 修改 (Modify)：按需求改代码
- 创造 (Make)：从零写

**数学领域 — CRA**
- 具象 (Concrete)：可操作的实物/模拟
- 表征 (Representational)：图示/图表
- 抽象 (Abstract)：符号/公式

**系统领域 — 分层追踪**
- 全景 → 单层 → 跨层追踪 → 系统整合

**隐性知识领域 — 认知学徒制**
- 专家建模 → 教练 → 脚手架 → 表达 → 反思 → 探索

**系统约束**：模块可选声明领域教学法。声明后，渲染引擎按该教学法的阶段序列组织内容，Agent 按该教学法的约束生成。

## 三、认知负荷是底盘，不是装饰

### 3.1 三种负荷

- **内在负荷**：主题本身的复杂度（元素交互性）。不能消灭，只能通过分解和排序来管理。
- **外在负荷**：糟糕的教学设计造成的。必须消灭：冗余信息、注意力分裂、装饰性元素。
- **相关负荷**：用于构建知识结构的认知资源。最大化它——当外在负荷降到零，剩下的工作记忆空间全给 schema 建构。

### 3.2 元素交互性决定教学策略

| 元素交互性 | 特征 | 策略 |
|-----------|------|------|
| 低 | 各概念可独立理解 | 直接讲解即可 |
| 中 | 2-3 个概念需同时处理 | Worked example + 分段 |
| 高 | 4+ 个概念互相依赖 | 先分解再整合，渐退脚手架 |

**系统约束**：模块应声明元素交互性等级。高交互性模块必须有 steps 块（分段原则），且不允许在一个 narrative 段落里引入超过 3 个新概念。

### 3.3 Worked Example 效应与渐退

研究反复确认：对新手，看 worked example 比自己解题学得更多更快。但对专家，worked example 反而是干扰（专家逆转效应）。

**渐退序列是脚手架和必要难度之间的桥梁**：

```
完整示范 → 半完成（最后几步省略）→ 大部分省略 → 自主解决
```

**系统约束**：
- 每个程序性知识模块至少有一个 worked example（steps 块）
- 练习系统必须支持多个脚手架级别（full / faded-1 / faded-2 / free）
- 同一个练习可以有多个变体，根据掌握度选择

## 四、难度不是敌人，该出现的难度必须出现

### 4.1 间隔效应（Spacing）

同一内容反复出现，间隔越来越长，记忆越持久。254 项研究的 meta-analysis 显示间隔练习比集中练习提高 10-30% 的留存率。

**系统约束**：课程级别必须有间隔复习策略。检索练习不只在当前模块末尾出现——隔 2 个模块后应再次出现（由课程 spacingSchedule 控制）。

### 4.2 交错效应（Interleaving）

混合不同类型的练习（而非集中练同一类型）提升区分能力。

**系统约束**：
- 模块内：教学用阻塞（集中练一个概念）
- 跨模块复习：用交错（混合多个模块的概念）
- 练习系统区分"学习模式"和"复习模式"

### 4.3 检索练习（Testing Effect）

主动从记忆中提取，比重新阅读有效 50%。检索本身是学习事件，不只是测量。

**系统约束**：
- 每个模块至少一个检索练习
- 检索练习标注 Bloom 层级
- "记忆"层检索最多占 30%，其余必须是"理解"以上
- 检索格式：生成优于选择（生成效应）——优先用 fill-gap / rebuild-map / explain，少用 multiple-choice

### 4.4 生成效应（Generation Effect）

自己产出答案比被给答案记得更牢。但前提是有足够背景知识——没有基础就要求生成是有害的。

**系统约束**：生成式练习只在学习者已完成该概念的"示范"和"脚手架练习"之后出现。

## 五、多媒体学习不是锦上添花

### 5.1 Mayer 的核心原则及系统映射

| 原则 | 含义 | 系统要求 |
|------|------|---------|
| 一致性 | 删除所有不服务学习目标的元素 | 每个叙事块标记 `isEssential`，非必要元素可折叠 |
| 信号 | 用视觉线索引导注意力 | steps 块的 highlight 字段，概念图的焦点标记 |
| 冗余 | 图 + 旁白 > 图 + 旁白 + 屏幕文字 | diagram 块不应重复相邻 text 块的内容 |
| 空间邻近 | 文字紧贴相关图示 | 叙事块可声明 `spatiallyLinkedTo` 关联 |
| 分段 | 用户控制节奏 | steps 块默认用户推进，不自动播放 |

### 5.2 图文不是两个独立的东西

**系统约束**：
- `diagram` 块永远不应单独出现——必须有相邻的 `text` 块解释
- `code` 块应与解释文本空间邻近，不应隔 3 个段落
- 概念图不是装饰，必须直接服务焦点问题

## 六、反思和元认知不是可选的

### 6.1 自我解释效应

让学习者解释"为什么这一步是这样"显著提升理解。不是加一句"你理解了吗"——是要求学习者产出解释。

### 6.2 元认知检查点

学习者需要监控自己的理解。系统应在关键节点提供：
- "停下来检查：你能不看材料解释 X 吗？"
- 如果卡住：提示重读哪个部分
- 如果顺畅：引导进入更难的挑战

**系统约束**：
- 概念性和策略性模块应至少有一个 `reflection` 叙事块
- 检索练习的 `answerShape` 字段应引导结构化思考，不只是给正确答案
