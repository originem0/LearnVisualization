# DESIGN.md — 可视化交互学习页面设计规范

> 每个模块页面的生成必须遵循以下原则。这不是建议，是硬性要求。

---

## 一、内容架构

1. **焦点问题** — 每个模块有且只有一个核心问题，所有内容围绕它展开
2. **反直觉开场** — 第一句话打破预期，制造认知冲突（编码优势）
3. **推导路径** — 不给定义，从问题出发一步步推出答案。读者跟着走发明者的思路
4. **悬念桥接** — 模块结尾留一个未解决的问题，自然导向下一个模块

## 二、可视化

5. **概念关系图** — 每个模块必须有一张 SVG 概念图（节点 + 箭头），展示所有核心概念的关系。读者先看全貌再深入
6. **对比图** — 有差异的概念必须视觉并排，不用文字描述差异（左/右、不同颜色）
7. **分步图示** — 复杂过程拆成离散的视觉步骤，每步只变化一个东西
8. **数据可视化** — 用颜色、大小、数量直接表达抽象概念，避免纯数字罗列

## 三、交互（核心差异化）

9. **自己动手试** — 提供输入框让读者用自己的数据测试学到的概念，立刻验证理解
10. **控制节奏的模拟器** — 用户点"下一步"控制过程推进，不是自动播放。这是认知负荷"分割"原则的交互实现
11. **即时反馈** — 操作后立刻看到结果变化，不需要提交、不需要等待
12. **预设示例按钮** — 降低"不知道输入什么"的门槛，示例本身就是教学内容

## 四、认知负荷控制

13. **减少冗余** — 能用图说清楚的不用文字，能用交互说清楚的不用图。文字只在必须解释"为什么"时出现
14. **渐进披露** — 详细/高级信息折叠，不一次性压垮读者
15. **交替实例** — 展示一个完成的例子，紧跟一个让用户自己操作的类似练习
16. **单一变量原则** — 每个交互组件只改变一个变量。对比时保持其他条件相同

## 五、工程约束

17. **客户端交互** — 交互组件用 `"use client"` + `dynamic(() => import(...), { ssr: false })`，其余保持静态生成
18. **零外部依赖** — 所有逻辑内置（模拟器、预计算数据），不引入新 npm 包
19. **暗色模式** — 全程用 CSS 变量（`--color-text`, `--color-panel`, `--color-border`, `--color-muted`），交互组件必须适配
20. **移动端适配** — 交互区域水平可滚动，按钮尺寸触屏友好，断点 `sm:` 以上才用多列布局

## 六、页面结构模板

每个模块页面的组成顺序：

```
1. Hero（标签 + 模块编号 + 标题 + 副标题）
2. 反直觉开场（视觉冲击 + 核心悬念）
3. 交互组件 A — "自己动手试"（让读者先体验再解释）
4. 概念关系图（SVG，展示本模块所有概念的关系全貌）
5. 叙事正文（推导路径，穿插：）
   - 对比图
   - 分步图示
   - 交互组件 B — 模拟器 / 步进演示
6. 悬念桥接（引出下一模块）
7. 折叠区 — 深入（概念列表、逻辑链、费曼测试、示例、反例）
8. 导航（上/下模块链接）
```

## 七、一句话原则

**概念图给全貌，交互组件让用户亲手验证，叙事文字只负责串联"为什么"。**

静态的部分尽量视觉化，能让用户动手的部分一定做成交互。
